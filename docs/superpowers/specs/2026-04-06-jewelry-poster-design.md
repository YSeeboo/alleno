# 饰品大图 设计文档

## 背景

订单需要生成饰品大图 PDF 供内部生产参考，每个饰品以大图+关键信息的方式展示，方便工人识别。

## 功能设计

### 入口

订单详情页饰品清单区域，新增【生成饰品大图】按钮。

### 输出格式

- **PDF 下载**：点击按钮直接下载 PDF
- **网页预览**：点击按钮旁的预览入口，在新标签页中打开预览

### PDF 布局

- 纸张：A4 纵向
- 网格：3 列 × 3 行 = 每页 9 个饰品
- 图片尽量占满格子空间，**不压缩图片质量**
- 每格内容（从上到下）：
  - 饰品图片（占格子 ≥ 80% 高度）
  - 文字区域（≤ 格子 20% 高度）：数量、单价（订单中的 unit_price）、客户货号（如有）、备注（如有）
- 饰品数量超过 9 个时自动分页
- 页面顶部显示：订单号 + 客户名

### 图片处理

- 从饰品的 `image` URL 下载原图
- **不做压缩/缩放质量降低**，仅按格子尺寸等比缩放适配
- 无图片的饰品显示占位框 + "暂无图片"文字

## API

### 新增接口

**GET `/orders/{order_id}/jewelry-poster-pdf`** — 下载饰品大图 PDF

```
Response: application/pdf (bytes)
```

**GET `/orders/{order_id}/jewelry-poster-preview`** — 网页预览（返回 HTML）

```
Response: text/html
```

## 后端实现

新建 `services/jewelry_poster.py`，使用 ReportLab 生成 PDF：

- A4 尺寸：595 × 842 pt
- 页边距：小（约 20pt），最大化图片区域
- 3×3 网格，每格约 185 × 255 pt
- 图片下载使用 httpx，保持原始质量
- 文字使用 STSong-Light 字体（与现有 PDF 一致）

网页预览可以返回一个简单的 HTML 页面，以 CSS Grid 实现 3×3 布局，图片使用原始 URL。

## 前端

### 订单详情页

在饰品清单卡片的 header-extra 区域，添加两个按钮：
- 【下载饰品大图】→ 调用 PDF 接口下载
- 【预览饰品大图】→ 新标签页打开预览 HTML
