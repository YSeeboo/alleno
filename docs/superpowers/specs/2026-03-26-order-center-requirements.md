# 订单中心需求规格

**日期:** 2026-03-26

**目标:** 将订单从简单的记录工具升级为生产调度的核心，串联电镀单、手工单，追踪成本和生产进度。

---

## 一、订单与生产关联

### 数据模型

**TodoList 配件行表 `order_todo_item`：**
- `id` (Integer, PK)
- `order_id` (FK → order.id)
- `part_id` (FK → part.id)
- `required_qty` (Numeric) — BOM 计算出的需求量

**关联中间表 `order_item_link`：**
- `id` (Integer, PK)
- `order_todo_item_id` (FK → order_todo_item.id, nullable) — 配件项关联 TodoList 行
- `order_id` (FK → order.id, nullable) — 饰品项直接关联订单
- `plating_order_item_id` (FK → plating_order_item.id, nullable) — 三选一
- `handcraft_part_item_id` (FK → handcraft_part_item.id, nullable) — 三选一
- `handcraft_jewelry_item_id` (FK → handcraft_jewelry_item.id, nullable) — 三选一

### 需求明细

| # | 需求 | 结论 |
|---|------|------|
| 1.1 | 关联方式 | 创建电镀/手工单时可关联 + 事后可关联/移除 |
| 1.2 | 关联粒度 | 配件项级别，不带数量。配件项关联 TodoList 配件行，饰品项直接关联 order_id |
| 1.3 | 关联对象 | 电镀配件项、手工配件项、手工饰品项、采购单配件项 |
| 1.4 | 关联唯一性 | 同一个电镀/手工配件项只能关联一个订单的 TodoList 行，不能同时关联多个订单 |
| 1.5 | 订单详情页 | 平铺展示所有关联的生产项及状态 |
| 1.6 | 电镀/手工单详情页 | 反向展示关联的订单号 |
| 1.7 | 订单列表页 | 显示生产进度概要（如"3/5 已完成"） |

### TodoList 生成与更新

| # | 需求 | 结论 |
|---|------|------|
| 1.8 | 生成时机 | 手动触发 — 订单详情页"生成配件清单"按钮 |
| 1.9 | 订单修改后更新 | 自动重新生成，保留能匹配的关联关系，只清除被删除的配件行 |

### 关联操作交互

| # | 需求 | 结论 |
|---|------|------|
| 1.10 | 单选关联 | 配件项旁边"关联订单"按钮 → 弹窗选订单 → 选 TodoList 中的配件行 |
| 1.11 | 批量关联 | 勾选多个配件项 → 点"批量关联订单" → 选订单 → 系统自动按配件(part_id)匹配 TodoList 行 |
| 1.12 | 批量匹配失败 | 跳过未匹配的配件，只关联能匹配的，提示未匹配项 |
| 1.13 | 解除关联 | 电镀/手工单详情页和订单详情页两边都可以操作解除 |

## 二、成本快照

### 成本公式

```
饰品单位成本 = Σ(配件.unit_cost × BOM.qty_per_unit) + 饰品.handcraft_cost
订单总成本 = Σ(饰品单位成本 × 数量) + packaging_cost
订单利润 = 订单总金额(售价) - 订单总成本
```

### 需求明细

| # | 需求 | 结论 |
|---|------|------|
| 2.1 | 触发时机 | 订单状态变为"已完成"时自动生成 |
| 2.2 | 前置校验 | 所有饰品必须有 BOM，否则阻止完成 |
| 2.3 | 饰品成本 | Σ(配件.unit_cost × BOM.qty_per_unit) + 饰品.handcraft_cost |
| 2.4 | 订单成本 | Σ(饰品单位成本 × 数量) + packaging_cost（包装费总额） |
| 2.5 | 利润 | 订单总金额 - 订单总成本 |
| 2.6 | 不可变性 | 快照生成后不随成本变化，退回状态时保留快照 |
| 2.7 | 饰品新增字段 | `handcraft_cost`（来自手工回收单饰品收回 price，通过 cost_sync 同步） |
| 2.8 | 订单新增字段 | `packaging_cost`（包装费总额，订单详情页随时可编辑） |

### 快照细节

| # | 需求 | 结论 |
|---|------|------|
| 2.9 | 快照内容 | 存完整明细：每个饰品的每个 BOM 配件的当时 unit_cost、用量、小计，便于回溯 |
| 2.10 | 配件无 unit_cost | 按 0 计算，允许完成但标记为不完整 |
| 2.11 | 快照展示 | 订单详情页增加"成本快照"板块，展示明细和利润 |
| 2.12 | handcraft_cost 同步 | 在手工回收单的 cost_sync 中实现，新增 `detect_handcraft_jewelry_cost_diffs`，手工回收单收回饰品时同步 price 到 `Jewelry.handcraft_cost` |

## 状态机更新

```
待生产 → 生产中 → 已完成（触发成本快照）
  ↓        ↓
已取消   已取消
```

前置条件：
- 待生产 → 生产中：无限制
- 生产中 → 已完成：所有饰品必须有 BOM
- 任意状态 → 已取消：允许，不影响关联的生产单
